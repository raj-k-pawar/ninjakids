## 🚀 Quick Start

### Prerequisites
- Flutter 3.19+ (`flutter --version`)
- Android Studio / VS Code
- Firebase account
- OpenAI account

### 1. Clone the repo
```bash
git clone https://github.com/your-username/ninjkids.git
cd ninjkids
```

### 2. Install dependencies
```bash
flutter pub get
```

### 3. Configure environment
```bash
cp .env.example .env
# Edit .env and add your OPENAI_API_KEY
```

### 4. Set up Firebase (see below)

### 5. Run code generation
```bash
flutter pub run build_runner build --delete-conflicting-outputs
```

### 6. Run the app
```bash
flutter run
```

## 🔥 Firebase Setup

### Step 1 — Create Firebase project
1. Go to [console.firebase.google.com](https://console.firebase.google.com)
2. Create a new project named **ninjkids-app**
3. Enable Google Analytics

### Step 2 — Enable services
In Firebase console, enable:
- **Authentication** → Email/Password + Google
- **Cloud Firestore** → Start in production mode
- **Firebase Storage**
- **Cloud Messaging**
- **Analytics**
- **Crashlytics**

### Step 3 — Register Android app
1. Package name: `com.ninjkids.app`
2. Download `google-services.json`
3. Place at `android/app/google-services.json`

### Step 4 — Use FlutterFire CLI (recommended)
```bash
# Install FlutterFire CLI
dart pub global activate flutterfire_cli

# Configure (auto-generates firebase_options.dart)
flutterfire configure
```

### Step 5 — Deploy Firestore rules
```bash
firebase deploy --only firestore:rules
```

## 🤖 OpenAI Setup

1. Get API key from [platform.openai.com](https://platform.openai.com)
2. Add to `.env`:
   ```
   OPENAI_API_KEY=sk-your-key-here
   ```
3. Pass at build time:
   ```bash
   flutter run --dart-define=OPENAI_API_KEY=sk-your-key
   ```

> **Cost tip:** GPT-4o-mini is used to keep costs low (~$0.0001 per question batch).
> For a student with 10 quizzes/day, cost ≈ **$0.001/day per kid**.

## 🏗 Build & Run

### Debug build
```bash
flutter run --dart-define=OPENAI_API_KEY=your-key
```

### Release APK
```bash
# 1. Create keystore (first time only)
keytool -genkey -v \
  -keystore android/app/ninjkids-release.jks \
  -alias ninjkids \
  -keyalg RSA -keysize 2048 -validity 10000

# 2. Create android/key.properties
echo "storePassword=your-store-pass
keyPassword=your-key-pass
keyAlias=ninjkids
storeFile=ninjkids-release.jks" > android/key.properties

# 3. Build APK
flutter build apk --release \
  --dart-define=OPENAI_API_KEY=your-key

# Output: build/app/outputs/flutter-apk/app-release.apk
```

### Release App Bundle (Play Store)
```bash
flutter build appbundle --release \
  --dart-define=OPENAI_API_KEY=your-key

# Output: build/app/outputs/bundle/release/app-release.aab
```

## ⚡ Codemagic CI/CD

### Setup
1. Connect your GitHub repo on [codemagic.io](https://codemagic.io)
2. Add environment variable groups:
   - `openai_credentials` → `OPENAI_API_KEY`
   - `firebase_credentials` → `GOOGLE_SERVICES_JSON` (base64)
   - `google_credentials` → `GCLOUD_SERVICE_ACCOUNT_CREDENTIALS`
3. Upload your keystore file and set `android_signing`
4. Push to `main` → automatic build triggers

### Pipelines
| Pipeline | Trigger | Output |
|---|---|---|
| `android-release` | Push to `main` | Release APK + AAB → Play Store |
| `android-debug` | Pull Request | Debug APK (QA testing) |

## 🏛 Architecture

NinjaKids follows **Clean Architecture** with **MVVM**:

```
Presentation Layer  →  Screens + Widgets + Riverpod Providers
     ↓ calls
Domain Layer        →  Use Cases + Entities + Repository Interfaces
     ↓ implements
Data Layer          →  Firebase + OpenAI + Local Cache (Hive)
```

**Data flow for AI Quiz:**
```
QuizScreen
  → aiQuestionServiceProvider
    → AiQuestionService.generateQuestions()
      → Fetch used question hashes from Firestore
      → Call OpenAI GPT-4o-mini with subject/class/difficulty
      → Parse JSON response into QuizQuestionEntity list
      → Deduplicate against history
      → Cache new question hashes in Firestore
    → return List<QuizQuestionEntity>
  → Display questions one by one with timer
  → On completion → AnalyticsService.saveQuizSession()
```

## 🗄 Firestore Schema

```
users/{userId}
  role: 'parent' | 'kid'
  email: string
  createdAt: timestamp

parents/{parentId}
  name, email, phone
  kidIds: string[]
  subscriptionPlan: 'free' | 'premium' | 'family'

kids/{kidId}
  parentId, name, avatarEmoji
  age, className
  pin: string (4 digits)
  allowedSubjects: string[]
  allowedGames: string[]
  dailyScreenTimeLimitMinutes: number
  todayScreenTimeMinutes: number
  isGamesLocked: boolean
  totalXp, level, coins
  currentStreak, longestStreak
  lastActiveAt: timestamp

quiz_history/{sessionId}
  kidId, subject, className
  score, total, accuracy
  xpEarned, coinsEarned
  timeTakenSeconds
  completedAt: timestamp

ai_questions/{kidId_subject}
  hashes: string[]          ← prevents duplicate AI questions
  updatedAt: timestamp

speaking_scores/{scoreId}
  kidId, language, lessonId
  pronunciationScore, fluencyScore, accuracyScore, overallScore
  completedAt: timestamp

achievements/{kidId_achievementId}
  kidId, achievementId
  isUnlocked: boolean
  unlockedAt: timestamp

screen_time_logs/{kidId_YYYY-MM-DD}
  kidId, date
  totalMinutes: number
  breakdown: { quiz: n, games: n, speaking: n }

subscriptions/{parentId}
  plan: 'free' | 'premium' | 'family'
  expiresAt: timestamp
  platform: 'android'
```

## 🔒 Security

- Parent controls protected by Firebase Auth
- Kids access via 4-digit PIN (stored hashed in Firestore)
- Firestore rules enforce parent-child relationship validation
- API keys passed via `--dart-define` (never hardcoded)
- flutter_secure_storage for sensitive local data
- No user data sold — child-safe COPPA-aware design
